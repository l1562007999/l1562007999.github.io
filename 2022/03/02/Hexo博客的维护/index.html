Hexo博客的维护

发表于2022-03-02|更新于2022-04-06|教程
|字数总计:469|阅读时长:1分钟|阅读量:

如果你已经根据网络教程创建好了你的Hexo博客, 你可以利用以下步骤来维护你的博客.

!!!提示: 你必须在node.js的命令行中或者giy bash中进入你的博客文件夹后 才能进行下面的命令行操作!!!

!!!如下命令行操作均基于windows平台!!!

维护步骤一: Create a new post//创建一篇博客

方法一: 通过本地目录 在source的_post中直接创建一个markdown文件.

方法二: 通过node.js command prompt这个命令行端口创建一个空的markdown文件.

接下来找到这个文件(blog/source/_post/), 这样你就可以在这个文件中编辑你的这篇博客内容了.

在markdown文件中写好你的博客之后, 接下来就可以在本地预览你的博客在网页中的呈现效果了.

维护步骤二: Generate static files// 生成静态博客网站

解释:你必须将改好的博客内容生成, 之后的预览功能才能完整显示

维护步骤三: Run server 利用localhost:4000本地运行以便预览你的博客网站

解释: 确保你的博客网站一切如预想的一样显示正常

操作: 在node.js的命令行中输入以下命令

接下来在浏览器中输入本地连接: http://localhost:4000/你就可以看到你的编辑好的博客网站了

维护步骤四:Deploy to remote sites 部署博客网站到远程仓库

将本地博客网站上传到你的github中以便通过网络分享给别人查看

部署成功后跨区通过网络网址查看你的博客吧!
